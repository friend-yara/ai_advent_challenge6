Demo: Memory Layering in Action
================================

Prerequisites
-------------
  cd /home/ya/ai_advent_challenge6
  source .venv/bin/activate

  # llm_agent is a symlink in ~/bin pointing to llm_agent_cli.py
  # No alias or python prefix needed.

Clean slate (remove any leftover state from previous sessions):
  rm -f state.toon short_term.toon


===========================================================================
STEP 1 — Long-term memory (LTM) effect on replies
===========================================================================

Goal: show that INVARIANTS.md constrains the answer (no SDK, use requests).

--- Session A: LTM enabled ---

  llm_agent \
    --model gpt-4.1-mini \
    --history-limit 6 \
    --context-strategy window \
    --use-profile \
    --use-project-memory \
    --use-invariants \
    --profile-file profiles/default/PROFILE.md \
    --project-memory-file profiles/default/PROJECT_MEMORY.md \
    --invariants-file profiles/default/INVARIANTS.md

Inside the REPL:
  > /task Implement retry logic for API calls
  > /state EXECUTION
  > /show
  > How should I implement retry with exponential backoff for HTTP requests?
  > /exit

Expected: answer mentions requests.post, raw HTTP, no openai SDK, no tenacity.

--- Session B: LTM disabled (clean short-term too) ---

  rm -f state.toon short_term.toon

  llm_agent \
    --model gpt-4.1-mini \
    --history-limit 6 \
    --context-strategy window

Inside the REPL:
  > How should I implement retry with exponential backoff for HTTP requests?
  > /exit

Expected: generic answer, may mention openai SDK, tenacity, or other libraries.

Conclusion: LTM constrains the reply in Session A; Session B is unconstrained.


===========================================================================
STEP 2 — Working memory survives restart (state.toon)
===========================================================================

Goal: task/state persists after exit; dialogue does not (no short_term.toon).

--- Session C: set task, save, exit ---

  rm -f state.toon short_term.toon

  llm_agent \
    --model gpt-4.1-mini \
    --history-limit 6 \
    --context-strategy window \
    --use-profile \
    --use-invariants

Inside the REPL:
  > /task Write a 3-step plan for adding a new CLI command
  > /state EXECUTION
  > /show
  > Tell me about step 1
  > /show
  > /save
  > /exit

--- Delete short-term only, restart ---

  rm -f short_term.toon

  llm_agent \
    --model gpt-4.1-mini \
    --history-limit 6 \
    --context-strategy window \
    --use-profile \
    --use-invariants

Inside the REPL:
  > /show

Expected:
  [working] task='Write a 3-step plan for adding a new CLI command', state=EXECUTION, step=0/0, current=''
  [stm]     messages=0, summary=no, facts=0, branch=main
  [ltm]     profile=yes, invariants=yes

  > Continue working on the task

Expected: agent knows the task from STATE block without being reminded.
  > /exit

Conclusion: working memory (state.toon) survives; dialogue is gone.


===========================================================================
STEP 3 — Short-term memory provides dialogue continuity
===========================================================================

Goal: show that short_term.toon restores conversation; losing it loses context.

--- Session D: establish dialogue ---

  rm -f state.toon short_term.toon

  llm_agent \
    --model gpt-4.1-mini \
    --history-limit 6 \
    --context-strategy window \
    --use-profile

Inside the REPL:
  > My name is Alexey and I work on the AI Advent Challenge
  > What do you know about me?
  > /show
  > /exit

Expected: agent knows name and project from the current dialogue window.

--- Restart WITH short_term.toon (keep it) ---

  llm_agent \
    --model gpt-4.1-mini \
    --history-limit 6 \
    --context-strategy window \
    --use-profile

Inside the REPL:
  > What do you know about me?

Expected: agent recalls name and project from short_term.toon.
  > /exit

--- Restart WITHOUT short_term.toon ---

  rm -f short_term.toon

  llm_agent \
    --model gpt-4.1-mini \
    --history-limit 6 \
    --context-strategy window \
    --use-profile

Inside the REPL:
  > What do you know about me?

Expected: agent has no context — replies something like
  "I don't have any previous messages in this session."
  > /exit

Conclusion: short_term.toon is the only source of dialogue continuity.


===========================================================================
STEP 4 — /ltm reload: update LTM without restart
===========================================================================

Goal: show that /ltm reload picks up file changes at runtime.

  llm_agent \
    --model gpt-4.1-mini \
    --history-limit 6 \
    --context-strategy window \
    --use-invariants

Inside the REPL:
  > /show
  (note [ltm] line)

  (in another terminal: edit profiles/default/INVARIANTS.md, e.g. add a line under Banned)

  > /ltm reload
  > /show
  (the updated content will be injected in the next prompt)
  > /exit


===========================================================================
What each test proves
===========================================================================

  Test                                         | What you observe
  ---------------------------------------------|--------------------------------------------
  Session A vs B (--use-invariants on/off)     | LTM constrains reply (no SDK vs. generic)
  Session C restart, no short_term.toon        | Task/state survive; dialogue is gone
  Session D restart with short_term.toon       | Dialogue continues; names recalled
  Session D restart without short_term.toon    | Dialogue forgotten; task not affected
  /ltm reload                                  | LTM updated in-process without restart
